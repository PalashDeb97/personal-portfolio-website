What is Git and why should you use Git?

Git is a free and open source version control system through which we can keep any code or work in one place, see any previous code if necessary, delete code, etc. Let's be a little more clear. Suppose you write 1 code today and 1 code tomorrow. If after 3 days you think, no I need that first day code. Generally speaking, can you figure out who was working on the same file first? No, I can't. But if we use Git we can easily extract any code any day. It means that this git is a tool for Version Control System of all our codes.


Why use Git?
Usually for a big project we have many developers working together and each one is given different tasks. Now, due to many developers working on the same project, so that there are no conflicts in the project, we use Git so that one person's code does not get mixed with another person's. It means that each person will work in a separate branch for his own work, due to which there will be no conflict. We will discuss about Branch below. Besides, the requirements of a project often change. For example, at the beginning of the project, there were these requirements, and after 7 days, another requirement, and then another requirement, but we wrote the code for each requirement. Using Git we can easily go back to any previous code. Again we work on several projects at the same time. In this case, Git plays a very important role for everyone's version control of the project.

What is JavaScript and its introduction.

JavaScript is a scripting or programming language that is commonly used in any website. This language is also called client-side scripting language because its source codes are processed through the client's web browser.
No web server is required to write or run the JavaScript language.
Server-side scripting languages ​​such as PHP and ASP allow you to write JavaScript code directly into the HTML code of your webpage. However, after writing the code of server-side scripting languages, when we see it in a Web Browser, it shows in an HTML form, that means if you look at the source code of your website, you can only see the HTML code, but in the case of JavaScript, it is a little different, it means that you can see what you wrote. . JavaScript was originally called LiveScript which was created in 1995 but later Netscape changed the name to JavaScript.


Advantages of JavaScript
There are some advantages to using JavaScript. For example-
1. The communication of your Web Application with the server is less - means through JavaScript you can validate the data of a form before sending it to the server whether all the data of this form is filled or not. This will reduce the traffic of your server which means the communication with the server will be less.
2. You can instantly show any message to the user. Means if the user forgets to fill something in the website you can immediately show a message. You don't have to wait for the entire page to load.
3. With JavaScript, you can create different interfaces that will work on different types of JavaScript events. 4. You can also create drag-and-drop components or various sliders with this JavaScript.
5. You can change the content of an HTML document through JavaScript.
6. You can change the value of HTML Attribute through JavaScript.
7. You can change the design that we design with CSS with JavaScript.
8. Easily show or hide any HTML Elements.
9. Can send request to any remote server, can download or upload any file (through AJAX).
10. And there are many more benefits...


Some Obligations of JavaScript
Like every programming language, JavaScript also has some restrictions that mean what you can't do with JavaScript. Let's find out a few-
1. You cannot read and write files through JavaScript. This feature is not available for security.
2. Cannot use JavaScript for any network related application.
3. There is no multi-threading or multiprocessor capability feature in JavaScript.


JavaScript Engine
At present, JavaScript can only be executed in the browser, but it can also be executed on any server or any other device. And for that only a JavaScript Engine is needed.
Web browsers already have a JavaScript engine, which is also called a JavaScript virtual machine. All these JavaScript Engines have different names in different Web Browsers. For example-
1. V8 (for Chrome and Opera browsers).
2. SpiderMonkey (for Firefox browser).
3. Trident and Chakra (for IE browser), ChakraCore (for Microsoft Edge browser).
4. Nitro and SquirrelFish (for Safari browser).


How does the JavaScript Engine work?
To tell how JavaScript Engine works, a lot needs to be said, but in general it can be said like this –
1. JavaScript Engine will first read your JavaScript code means parse it.
2. Then your JavaScript code will be changed and compiled into a machine language.
3. Then this machine language will work for you.

What is HTML? How does HTML work?

HTML is a markup language with which you can create a web site. The full meaning of HTML is: HyperText Markup Language.Here HyperText means a method by which you can go from one website to another on the web. And to go from one website to another, you have to click on any text. And this clicked text is called hyperlinks. Which will take you to another website. Markup is the HTML tags that you write in your text files. With these tags you determine how the text of your file will look like. For example: Bold or Italic etc.


How does HTML work?
Basically HTML is made up of some short codes which are written in a file and these short codes are called tags. Then save this file with a name and file that you can later view with any browser. For example, with Internet Explorer or Chrome browser. When you open this file in a browser, the browser will read the codes in your file and process it and show you a text version of it. Upteen can write these tags in any code editor like Nodepad++, Sublime Text 3, Atom etc.

What is CSS? What is CSS?

CSS stands for Cascading Style Sheets CSS and is a design language. Means we use this CSS to work related to website design. Through this we determine the style of any HTML document of our website. For example, what will a box look like, what will its color be, what will be its length or width etc. We even use CSS to dictate how each HTML tag will look on our website.
When we create a website, we first write the markup or HTML code. But if no style is given in these HTML codes then the HTML file does not get fullness or it is not like our design or it will look like the way the HTML codes are written. But if we want the HTML file to look like our design then we must use CSS. As a result, we will be able to define the color, font size, etc. of any tag as desired. We can use CSS to tell how much space a part will take up or in which direction the part will show, etc. For example, suppose we want to build a house. For that we need bricks, sand, cement, rods. These bricks, sand, cement, rods etc. are the HTML tags through which we can build the structure of our house. But if we want to have this color on this side of the house, there will be a picture on the wall, there will be a border around the picture, one side of the whole house will be blue color etc. This is the main work of CSS.
